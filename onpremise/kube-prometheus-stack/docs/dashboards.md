# Grafana Dashboard Guide

Custom and import dashboard management guide.

<br/>

## Custom Dashboards

Managed as JSON files in `dashboards/` directory.

| File | Target |
|------|--------|
| `mysql-dashboard.json` | MySQL (Connections, QPS, InnoDB, Slow Query) |
| `redis-dashboard.json` | Redis (Memory, Commands, Keys, Hit Rate) |
| `fluentbit-fluentd-dashboard.json` | Fluent Bit + Fluentd logging pipeline |
| `ingress-nginx-dashboard.json` | Ingress-Nginx controllers (mgmt + public-a~j) |
| `argocd-dashboard.json` | ArgoCD (App sync, Git requests, Cluster status) |
| `harbor-dashboard.json` | Harbor (Projects, Storage, HTTP requests) |

### Import Methods

**Method 1: Grafana UI**

1. Grafana → **Dashboards** → **New** → **Import**
2. Click **Upload JSON file**
3. Select JSON from `dashboards/` directory
4. Data source: **Prometheus** → Import

**Method 2: Grafana API**

```bash
cd observability/monitoring/kube-prometheus-stack

# Fluent Bit + Fluentd
cat dashboards/fluentbit-fluentd-dashboard.json | \
  python3 -c "import sys,json; d=json.load(sys.stdin); print(json.dumps({'dashboard':d,'overwrite':True}))" | \
  curl -X POST http://grafana.example.com/api/dashboards/db \
    -H "Content-Type: application/json" \
    -u admin:<password> -d @-

# Import all dashboards at once
for f in dashboards/*.json; do
  echo "Importing: $(basename $f)"
  cat "$f" | \
    python3 -c "import sys,json; d=json.load(sys.stdin); print(json.dumps({'dashboard':d,'overwrite':True}))" | \
    curl -s -X POST http://grafana.example.com/api/dashboards/db \
      -H "Content-Type: application/json" \
      -u admin:<password> -d @- | python3 -c "import sys,json; print(json.load(sys.stdin).get('status','error'))"
done
```

## Import Dashboards (Grafana.com)

Grafana → **Dashboards** → **New** → **Import** → Enter ID → Data source: **Prometheus** → Import

| Target | Dashboard ID | Name |
|--------|-------------|------|
| K8s Nodes / Physical Servers / VMs | `1860` | [Node Exporter Full](https://grafana.com/grafana/dashboards/1860) |
| MySQL | `14057` | [MySQL Overview](https://grafana.com/grafana/dashboards/14057) |
| Redis | `11835` | [Redis Dashboard](https://grafana.com/grafana/dashboards/11835) |
| ArgoCD | `14584` | [ArgoCD](https://grafana.com/grafana/dashboards/14584) |
| Harbor | `14930` | [Harbor](https://grafana.com/grafana/dashboards/14930) |
| Ingress-Nginx (alt) | `9614` | [NGINX Ingress Controller](https://grafana.com/grafana/dashboards/9614) |

> K8s default dashboards (Pod, Namespace, Workload, etc.) are auto-generated by kube-prometheus-stack.

## Dashboard UID List

Each dashboard has a unique `uid`. Re-importing with `overwrite: true` updates existing dashboards without creating duplicates.

| Dashboard | UID | Variables (Dropdown) |
|-----------|-----|---------------------|
| MySQL Overview | `mysql-overview` | `job`, `instance` |
| Redis Overview | `redis-overview` | `job`, `instance` |
| Logging Pipeline | `logging-pipeline-fluent` | `fb_job`, `fb_instance`, `fd_job`, `fd_instance` |
| Ingress-Nginx | `ingress-nginx-controller` | `controller`, `namespace`, `ingress` |
| ArgoCD Overview | `argocd-overview` | `job`, `namespace` |
| Harbor Registry | `harbor-registry` | `job`, `instance` |

> UID visible in Grafana URL: `http://grafana.example.com/d/<UID>/...`

<br/>

## Dashboard Backup/Export

```bash
# Export specific dashboard by UID
curl -s http://grafana.example.com/api/dashboards/uid/<UID> \
  -u admin:<password> | python3 -m json.tool > dashboards/<filename>.json
```
