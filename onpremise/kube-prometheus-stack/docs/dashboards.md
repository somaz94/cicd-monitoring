# Grafana Dashboard Guide

Custom and import dashboard management guide.

<br/>

## Custom Dashboards

Managed as JSON files in `dashboards/` directory.

| File | Target |
|------|--------|
| `mysql-dashboard.json` | MySQL (Connections, QPS, InnoDB, Slow Query) |
| `redis-dashboard.json` | Redis (Memory, Commands, Keys, Hit Rate) |
| `fluentbit-fluentd-dashboard.json` | Fluent Bit + Fluentd logging pipeline |
| `nginx-gateway-dashboard.json` | NGINX Gateway Fabric — control-plane (reconcile loop, work queue, NGF event batch, resources) + data-plane (request rate, connection state, accept/handle rate, agent CPU
| `argocd-dashboard.json` | ArgoCD (App sync, Git requests, Cluster status) |
| `harbor-dashboard.json` | Harbor (Projects, Storage, HTTP requests) |

> The former `ingress-nginx-dashboard.json` has moved to `dashboards/_deprecated/` and is no longer imported automatically. It was retired after the 2026-04-17 ingress-nginx → NGF cutover.

### Import Methods

**Method 1: Grafana UI**

1. Grafana → **Dashboards** → **New** → **Import**
2. Click **Upload JSON file**
3. Select JSON from `dashboards/` directory
4. Data source: **Prometheus** → Import

**Method 2: `scripts/import-dashboards.sh` (recommended)**

POSTs every file under `dashboards/*.json` to the Grafana HTTP API. Re-runs are idempotent because dashboards with the same UID are overwritten (`overwrite: true`).

```bash
cd observability/monitoring/kube-prometheus-stack

# Bulk import — password resolved from the in-cluster secret
./scripts/import-dashboards.sh --all --from-secret

# Bulk import — password via environment variable
GRAFANA_PASSWORD=<password> ./scripts/import-dashboards.sh --all

# Specific files (repeat -f)
./scripts/import-dashboards.sh -f dashboards/fluentbit-fluentd-dashboard.json -p <password>

# Skip specific files (substring match, comma-separated)
./scripts/import-dashboards.sh --all --except ingress-nginx,metallb --from-secret

# Dry-run
./scripts/import-dashboards.sh --all --dry-run
```

Key flags:

| Flag | Description |
|---|---|
| `-f, --file PATH` | Single file. Repeat `-f` for multiple. |
| `--all` | Every `*.json` directly under `dashboards/` (default when `-f` is absent). |
| `--except PAT[,...]` | With `--all`, exclude files whose basename contains any substring. |
| `-p, --password PASS` | Password (or `GRAFANA_PASSWORD` env). |
| `--from-secret` | Read `monitoring/kube-prometheus-stack-grafana` key `admin-password` via `kubectl`. |
| `-u, --url URL` | Grafana base URL (default: `http://grafana.example.com`). |
| `-U, --user USER` | Grafana user (default: `admin`). |
| `-n, --dry-run` | Print targets only; no POST. |
| `-v, --verbose` | Print full Grafana response body per file. |

Run `./scripts/import-dashboards.sh --help` for the full option list.

> Only JSON files placed directly under `dashboards/` are imported. Sub-directories such as `dashboards/_deprecated/` are skipped automatically — move deprecated dashboards there to retire them without losing the file.

Grafana → **Dashboards** → **New** → **Import** → Enter ID → Data source: **Prometheus** → Import

| Target | Dashboard ID | Name |
|--------|-------------|------|
| K8s Nodes / Physical Servers / VMs | `1860` | [Node Exporter Full](https://grafana.com/grafana/dashboards/1860) |

## Dashboard UID List

Each dashboard has a unique `uid`. Re-importing with `overwrite: true` updates existing dashboards without creating duplicates.

| Dashboard | UID | Variables (Dropdown) |
|-----------|-----|---------------------|
| MySQL Overview | `mysql-overview` | `job`, `instance` |
| Redis Overview | `redis-overview` | `job`, `instance` |
| Logging Pipeline | `logging-pipeline-fluent` | `fb_job`, `fb_instance`, `fd_job`, `fd_instance` |
| NGINX Gateway Fabric | `nginx-gateway-fabric` | `namespace`, `pod`, `controller` |
| ArgoCD Overview | `argocd-overview` | `job`, `namespace` |
| Harbor Registry | `harbor-registry` | `job`, `instance` |
| (deprecated) Ingress-Nginx | `ingress-nginx-controller` | file: `dashboards/_deprecated/ingress-nginx-dashboard.json` |

> UID visible in Grafana URL: `http://grafana.example.com/d/<UID>/...`

<br/>

## Dashboard Backup/Export

```bash
# Export specific dashboard by UID
curl -s http://grafana.example.com/api/dashboards/uid/<UID> \
  -u admin:<password> | python3 -m json.tool > dashboards/<filename>.json
```
